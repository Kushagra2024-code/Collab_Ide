# Prompt: Add AI-Powered Collaborative Development Workspace

I already have a collaborative coding platform. Extend the existing project by adding the following features without breaking the current architecture. Keep the code modular, scalable, secure, and production-ready.

## 1. Collaborative File Explorer

Implement a complete project file management system.

Features:

* Tree-based file and folder explorer.
* Create, rename, move, copy, delete files and folders.
* Drag-and-drop support.
* Upload and download files.
* Search files.
* Recent files.
* Favorite files.
* File icons based on extension.
* File history.
* Auto refresh when another user changes files.
* Permission-aware operations.

## 8. AI Coding Agent (Powered by Gemini)

Integrate Google's Gemini model as an AI development assistant that understands the entire project.

The AI should have read access to the complete project and act as an assistant. It must **never make changes automatically**. Every modification requires explicit user approval.

Capabilities:

* Understand the entire codebase.
* Explain project architecture.
* Explain files, folders, classes, functions, and APIs.
* Generate documentation.
* Summarize modules.
* Detect bugs.
* Find unused code.
* Detect dead code.
* Suggest refactoring.
* Improve code quality.
* Explain compiler/runtime errors.
* Suggest fixes.
* Review pull requests.
* Generate unit tests.
* Generate integration tests.
* Explain algorithms.
* Explain design patterns.
* Recommend best practices.
* Generate README sections.
* Explain database schema.
* Generate API documentation.
* Suggest performance improvements.
* Detect security vulnerabilities.
* Detect duplicate code.
* Recommend project structure improvements.
* Generate SQL queries.
* Help with Git operations.
* Generate commit messages.
* Explain terminal commands.
* Help configure Docker.
* Help configure CI/CD.
* Answer project-specific questions using the current repository context.

Agent behavior:

* Never modify code without user approval.
* Show a preview/diff before any change.
* Explain why the change is recommended.
* Allow the user to accept, reject, or edit the suggestion.
* Keep a history of AI suggestions.
* Support conversational memory within the project session.
* Reference relevant files and line numbers in explanations.

---

## 9. AI Code Modification Workflow

Workflow:

1. User requests a change.
2. AI analyzes the repository.
3. AI proposes a solution.
4. AI displays a diff preview.
5. User approves or rejects.
6. Only after approval are changes applied.
7. Every AI-generated change is logged.
